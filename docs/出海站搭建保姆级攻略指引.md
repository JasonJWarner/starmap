## 1. 项目概述

### 1.1 什么是 StarMap

StarMap 是一个面向开发者社区的 GitHub 优质开源项目导航站，通过 GitHub Search API 定期抓取、清洗、分类 2000+ 优质开源项目，帮助开发者快速找到需要的工具和资源。

**核心数据：**
- 收录项目：2462 个（持续更新）
- 分类数量：10 个一级分类
- 更新频率：每周自动同步
- 部署平台：Vercel（免费）
- 域名：starmap-nav.vercel.app（可绑定自定义域名）

### 1.2 10 个一级分类

| 分类 | 说明 | 典型项目 |
|------|------|---------|
| AI工具与模型 | 大模型、AI Agent、向量数据库等 | GPT4All、LangChain、Ollama |
| 实用软件与桌面应用 | 跨平台桌面软件 | Obsidian、VS Code |
| 数据与API | 公开数据集、API 接口 | public-apis、HuggingFace Datasets |
| 开发工具与IDE | 开发辅助、CLI 工具 | Neovim、Helix、Git |
| 自动化与效率工具 | 自动化脚本、workflow 工具 | n8n、Home Assistant |
| 学习教程与课程 | 编程学习资源 | freeCodeCamp、The Odin Project |
| Awesome 精选导航 | 各类 awesome 列表汇总 | awesome-python、awesome-machine-learning |
| 设计素材与图标 | 设计工具、UI 组件 | Figma、Lucide |
| 开源替代品 | 闭源软件的开源替代 | LibreOffice、GIMP |
| 网站模板与建站资源 | CMS、静态网站生成器 | Hugo、Jekyll |

### 1.3 技术栈

## 4. 第一步：GitHub 数据采集

### 4.1 采集原理

利用 GitHub Search API 按关键词搜索项目：
```
https://api.github.com/search/repositories?q=关键词&sort=stars&order=desc&per_page=100
```

**关键参数说明：**
- `q`：搜索查询，支持多种过滤条件
- `stars:>3000`：只搜索 3000+ stars 的项目（保证质量）
- `pushed:>2025-10-19`：只搜索近 6 个月有更新的（保证活跃度）
- `sort=stars`：按 stars 降序排列
- `per_page=100`：每页最多 100 条（API 上限）

### 4.2 创建采集脚本

以 AI 工具分类为例（`fetch-ai-repos-batch1.js`）：

```javascript
const https = require('https');
const fs = require('fs');
const path = require('path');

const MIN_STARS = 3000;
const SIX_MONTHS_AGO = '2025-10-19';
const OUT_FILE = path.join(__dirname, '..', 'data', 'ai-tools-models-batch1.json');

// 定义多个关键词查询，覆盖同一分类下的不同子主题
const QUERIES = [
  { q: `LLM stars:>${MIN_STARS} pushed:>${SIX_MONTHS_AGO}`, sub: '大语言模型' },
  { q: `GPT model stars:>${MIN_STARS} pushed:>${SIX_MONTHS_AGO}`, sub: 'GPT/ChatGPT' },
  { q: `diffusion model stars:>${MIN_STARS} pushed:>${SIX_MONTHS_AGO}`, sub: 'Diffusion/图像生成' },
  { q: `stable diffusion stars:>${MIN_STARS} pushed:>${SIX_MONTHS_AGO}`, sub: 'Stable Diffusion' },
  { q: `AI agent stars:>${MIN_STARS} pushed:>${SIX_MONTHS_AGO}`, sub: 'AI Agent' },
  { q: `RAG stars:>${MIN_STARS} pushed:>${SIX_MONTHS_AGO}`, sub: 'RAG/检索增强' },
  { q: `vector database stars:>${MIN_STARS} pushed:>${SIX_MONTHS_AGO}`, sub: '向量数据库' },
  { q: `local LLM stars:>${MIN_STARS} pushed:>${SIX_MONTHS_AGO}`, sub: '本地部署LLM' },
];

function fetchGitHub(query) {
  return new Promise((resolve, reject) => {
    const encodedQ = encodeURIComponent(query);
    const url = `https://api.github.com/search/repositories?q=${encodedQ}&sort=stars&order=desc&per_page=100`;
    const options = {
      headers: {
        'User-Agent': 'GitHub-Resource-Discovery/1.0',
        'Accept': 'application/vnd.github.v3+json',
        // 如果申请了 Token，在这里加上：
        // 'Authorization': 'token ' + process.env.GITHUB_TOKEN
      }
    };
    https.get(url, options, (res) => {
      let data = '';
      res.on('data', chunk => data += chunk);
      res.on('end', () => {
        try { resolve(JSON.parse(data)); }
        catch { resolve({ items: [] }); }
      });
    }).on('error', reject);
  });
}

async function sleep(ms) { return new Promise(r => setTimeout(r, ms)); }

async function main() {
  let repos = [];

  for (const { q, sub } of QUERIES) {
    console.log(`Fetching: ${sub} ...`);
    const result = await fetchGitHub(q);
    const items = result.items || [];
    console.log(`  Got ${items.length} repos`);

    for (const repo of items) {
      // 过滤条件：排除 archived、fork 数为 0、无描述的项目
      if (repo.archived || repo.forks_count === 0 || !repo.description) continue;
      // 防止重复
      if (!repos.find(r => r.name === repo.full_name)) {
        repos.push({
          name: repo.full_name,
          description: repo.description.replace(/[\r\n\t]/g, ' ').trim(),
          stars: repo.stargazers_count,
          forks: repo.forks_count,
          language: repo.language || '',
          topics: (repo.topics || []).join(','),
          url: repo.html_url,
          pushedAt: repo.pushed_at,
          subCategory: sub,
          homepage: repo.homepage || ''
        });
      }
    }

    // 每查询完一个关键词就保存一次（防止脚本中断数据丢失）
    fs.writeFileSync(OUT_FILE, JSON.stringify(repos, null, 2), 'utf-8');
    await sleep(7000); // GitHub API 限制：两次请求间隔至少 7 秒
  }

  console.log(`\nDone: ${repos.length} repos saved`);
}

main().catch(e => { console.error(e); process.exit(1); });
```

### 4.3 关键词设计原则

采集质量直接取决于关键词覆盖范围，设计原则如下：

**原则 1：一个分类用多个关键词**
- 单一关键词漏采率很高（GitHub 索引对中文关键词支持差）
- 每个关键词定义一个 `subCategory`（子分类）
- 建议每个分类 8~15 个关键词

**原则 2：关键词要具体不要泛**
- ✅ `python web framework` → 比 `python` 更精准
- ✅ `local LLM` → 专门找本地部署方案
- ❌ `machine learning` → 太泛，返回结果质量参差不齐

**原则 3：过滤 archived/stale 项目**
- GitHub 上有大量 archived 项目但 stars 很高
- 加上 `NOT archived` 过滤（但 rate limit 会更高）
- 或者在代码中 `if (repo.archived) continue;`

**原则 4：设置 stars 下限**
- `stars:>3000` 是合理起点，太低质量差，太高数量少
- 热门分类（AI/开发工具）可提高到 `stars:>5000`

## 5. 第二步：数据清洗与合并

### 5.1 合并所有分类数据

`merge-all.js` 读取 `data/` 目录下所有分类 JSON，合并到 `master-catalog.json`：

```javascript
const fs = require('fs');
const path = require('path');

const DATA_DIR = path.join(__dirname, '..', 'data');
const CATEGORY_FILES = [
  'ai-tools-models.json',
  'automation-efficiency.json',
  'data-api.json',
  'dev-tools-ide.json',
  'desktop-apps.json',
  'awesome-lists.json',
  'design-resources.json',
  'learning-tutorials.json',
  'open-source-alternatives.json',
  'website-templates.json',
];

function loadJson(filename) {
  const filePath = path.join(DATA_DIR, filename);
  if (!fs.existsSync(filePath)) return [];
  const raw = fs.readFileSync(filePath, 'utf-8');
  try { return JSON.parse(raw); } catch { return []; }
}

function main() {
  const allRepos = [];

  for (const file of CATEGORY_FILES) {
    const repos = loadJson(file);
    const category = file.replace('.json', '');
    for (const repo of repos) {
      repo.category = category;
      allRepos.push(repo);
    }
  }

  // 去重（按 repo.name）
  const seen = new Set();
  const unique = allRepos.filter(r => {
    if (seen.has(r.name)) return false;
    seen.add(r.name);
    return true;
  });

  fs.writeFileSync(
    path.join(DATA_DIR, 'master-catalog.json'),
    JSON.stringify(unique, null, 2),
    'utf-8'
  );
  console.log(`Merged ${allRepos.length} → ${unique.length} unique repos`);
}

main();
```

### 5.2 生成站点用优化 JSON

`prepare-site-data.js` 将 master-catalog.json 压缩为站点专用格式（字段精简）：

```javascript
const fs = require('fs');
const path = require('path');

const master = JSON.parse(
  fs.readFileSync(path.join(__dirname, '..', 'data', 'master-catalog.json'), 'utf-8')
);

const CATEGORY_MAP = {
  'ai-tools-models': 'AI工具与模型',
  'automation-efficiency': '自动化与效率工具',
  'data-api': '数据与API',
  'dev-tools-ide': '开发工具与IDE',
  'desktop-apps': '实用软件与桌面应用',
  'awesome-lists': 'Awesome精选导航',
  'design-resources': '设计素材与图标',
  'learning-tutorials': '学习教程与课程',
  'open-source-alternatives': '开源替代品',
  'website-templates': '网站模板与建站资源',
};

const categories = {};
for (const [key, name] of Object.entries(CATEGORY_MAP)) {
  categories[key] = {
    name,
    repos: []
  };
}

for (const repo of master) {
  const cat = repo.category;
  if (!categories[cat]) continue;
  // 精简字段，只保留站点需要的
  categories[cat].repos.push({
    n: repo.name,           // name
    d: repo.description,    // description
    s: repo.stars,          // stars
    sf: formatStars(repo.stars), // stars formatted
    l: repo.language,       // language
    t: repo.topics,         // tags
    sc: repo.subCategory,   // subCategory
    u: repo.url,            // GitHub URL
    h: repo.homepage,       // homepage
    p: repo.pushedAt        // pushed date
  });
}

const siteData = {
  version: '1.0',
  updatedAt: new Date().toISOString().split('T')[0],
  totalRepos: master.length,
  categories
};

fs.writeFileSync(
  path.join(__dirname, '..', 'site', 'data', 'catalog.json'),
  JSON.stringify(siteData),
  'utf-8'
);
console.log(`Site data ready: ${master.length} repos`);
```

```javascript
// app.js 路由核心
function handleRoute() {
  const hash = window.location.hash || '#/';

  if (hash === '#/' || hash === '') {
    renderHome();
  } else if (hash.startsWith('#/category/')) {
    const category = hash.replace('#/category/', '');
    renderCategory(category);
  } else if (hash.startsWith('#/search/')) {
```

### 6.5 难度标签体系设计

**设计思路：**
- 难度标签分三级：入门🟢 / 进阶🟡 / 专业🔴
- 特殊标记：偏门⚠️ / 商业💰
- 标签存储在 `tags.json`（独立于 catalog.json，方便更新）

**难度自动标注规则（启发式）：**
```javascript
function assignDifficulty(repo) {
  const desc = (repo.d || '').toLowerCase();
  const name = (repo.n || '').toLowerCase();
  const combined = desc + ' ' + name;

  // awesome 列表类 → 专业（内容是高级参考）
  if (name.includes('awesome-') && repo.s > 15000) return '专业';

  // star 阈值判断
  if (repo.s > 50000) return '专业';
  if (repo.s > 15000) return '进阶';

  // 向量数据库/框架类 → 降一档（使用简单）
  if (combined.includes('vector database') || combined.includes('ollama')) {
    return repo.s > 15000 ? '进阶' : '入门';
  }

  return '入门';
}
```

```javascript
function projectCard(repo, tags) {

  return `
    <div class="project-card ${isLocked ? 'locked' : ''}">
      ${isLocked ? `<div class="blur-overlay"></div>` : ''}
      <!-- 卡片内容 -->
      ${isLocked ? `<button class="unlock-btn" data-repo="${repo.n}">🔓 解锁查看</button>` : ''}
    </div>
  `;
}

// 解锁按钮点击事件
document.addEventListener('click', async (e) => {
  if (e.target.classList.contains('unlock-btn')) {
    const repoName = e.target.dataset.repo;
    e.target.closest('.project-card').classList.remove('locked');
    // 记录到 Supabase
    await trackUnlock(repoName);
  }
});
```

## 8. 第五步：部署上线

### 8.1 GitHub 创建仓库

1. 登录 GitHub → 点击右上角 `+` → "New repository"
2. 仓库名：`starmap`
3. 选择 Public（Vercel 免费版要求 Public 仓库）
4. 不要勾选 "Initialize with README"
5. 点击 "Create repository"

### 8.2 本地初始化并推送

```powershell
cd C:\path\to\starmap\site
git init
git add -A
git commit -m "feat: initial commit"
git branch -M main
git remote add origin git@github.com:YourUsername/starmap.git
git push -u origin main
```

> ⚠️ 如果推送时出现 `ssh: connect to host github.com port 22: Connection timed out`，说明网络封锁了 SSH 22 端口。需要配置 SSH 通过 HTTPS 代理：
> ```powershell
> # 在 C:\Users\YourName\.ssh\config 中添加：
> Host github.com
>   Hostname github.com
>   User git
>   Port 443
>   ProxyCommand connect -S 127.0.0.1:10808 %h %p
> ```

### 8.3 Vercel 部署

**方式 1：通过 Vercel 网站导入**

1. 登录 https://vercel.com
2. 点击 "Add New" → "Project"
3. 授权 GitHub，选择 `starmap` 仓库
4. Framework Preset 选择 "Other"
5. Build Command 留空，Output Directory 填 `.`
6. 点击 "Deploy"

**方式 2：Vercel CLI（用于手动触发部署）**

```powershell
npm install -g vercel

vercel --token YourToken --yes
```

Token 在 Vercel Dashboard → Settings → Tokens 创建。

**方式 3：自动部署（推荐）**

在 Vercel Dashboard 中启用 GitHub Integration：
1. Project → Settings → Git → Connect GitHub
2. 选择仓库后，每次 `git push` 自动触发部署

### 8.4 域名绑定（可选）

1. Vercel Dashboard → Project → Settings → Domains
2. 输入自定义域名（如 `starmap.dev`）
3. 按提示在域名 DNS 处添加记录（CNAME 指向 `cname.vercel-dns.com`）

### 8.5 环境变量（敏感信息）

如果使用 Supabase 等后端服务，敏感 Key 不要提交到 GitHub：
- Vercel Dashboard → Project → Settings → Environment Variables
- 添加 Key-Value 对，部署时自动注入

## 9. 第六步：数据监控与分析

### 9.1 Umami（网站整体流量）

**申请与配置：**
1. 注册 https://cloud.umami.is
2. 创建一个 Website，获取 Website ID
3. 在 index.html 中加入：

```html
<script defer src="https://cloud.umami.is/script.js"

```

**优点：**
- 隐私友好（GDPR 合规，不使用 Cookie）
- 免费计划每月 10 万 PV
- 提供实时访问、页面浏览量、跳出率等基础指标

### 9.2 Supabase（用户行为数据）

**申请与配置：**
1. 注册 https://supabase.com
2. 创建项目，获取 Project URL 和 `anon/public` key
3. 创建 `unlock_events` 表：
```sql
CREATE TABLE unlock_events (
  id uuid DEFAULT gen_random_uuid() PRIMARY KEY,
  created_at timestamptz DEFAULT now(),
  repo_name text NOT NULL,
  repo_category text,
  user_fingerprint text NOT NULL
);
```

**RLS 策略（允许插入，禁止读取）：**
```sql
ALTER TABLE unlock_events ENABLE ROW LEVEL SECURITY;
CREATE POLICY "Allow anon insert" ON unlock_events
  FOR INSERT TO anon WITH CHECK (true);
```

**前端埋点代码：**
```javascript
async function trackUnlock(repoName, repoCategory) {
  const fp = getFingerprint(); // 从 localStorage 读取或生成
  await fetch(`${SUPABASE_URL}/rest/v1/unlock_events`, {
    method: 'POST',
    headers: {
      'Content-Type': 'application/json',
      'apikey': SUPABASE_ANON_KEY,
      'Authorization': `Bearer ${SUPABASE_ANON_KEY}`
    },
    body: JSON.stringify({
      repo_name: repoName,
      repo_category: repoCategory,
      user_fingerprint: fp
    })
  });
}
```

## 10. 第七步：持续运营与维护

### 10.1 数据同步流程

```
每周日 03:00（北京时间）
    ↓
master-sync.js 串行执行 11 个采集脚本
    ↓（约 25~35 分钟）
merge-all.js 合并所有分类数据 → master-catalog.json
    ↓
prepare-site-data.js 压缩为 site/data/catalog.json
    ↓
git add + commit + push
    ↓
Vercel 自动部署（GitHub webhook 触发）
```

### 10.2 数据质量检查

定期检查以下指标：
- catalog.json 中的项目数量（是否大幅下降）
- tags.json 覆盖率（应为 100%）
- 有无 archived/stale 项目混入
- 分类分布是否合理

检查脚本：
```javascript
// scripts/check-data.js
const catalog = require('../site/data/catalog.json');
const tags = require('../data/tags.json');

console.log('Total repos:', catalog.totalRepos);
console.log('Tags coverage:', Object.keys(tags).length, '/', catalog.totalRepos);
```

### 10.3 常见问题排查

| 问题 | 排查方向 |
|------|---------|
| Vercel 没自动部署 | 检查 GitHub webhook 是否触发，Vercel 项目是否连接了 GitHub |
| GitHub API 返回 401 | Token 过期或未配置，重新生成 Token |
| Git push 超时 | 网络问题，尝试配置 SSH 代理或使用 HTTPS |
| 采集数据减少 | GitHub API 限速、关键词匹配变化、archived 项目被过滤 |
| 页面加载慢 | catalog.json 过大（>1.5MB），检查 prepare-site-data.js 压缩比 |
